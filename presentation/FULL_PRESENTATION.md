## 1. Обзор системы

### Что такое Scoliologic Wiki?

**Scoliologic Wiki** — это современная корпоративная платформа для управления знаниями, разработанная специально для группы компаний Scoliologic.

Система объединяет:
- **Удобство Notion** — интуитивный редактор с полным набором инструментов
- **Мощь AI** — локальные AI-модели для поиска, анализа и генерации текста
- **Безопасность** — полная конфиденциальность данных на собственном сервере
- **Масштабируемость** — готовность к работе с большими объёмами контента

### Ключевые преимущества

✅ **Локальный AI** — все данные остаются на вашем сервере  
✅ **Простота использования** — не требует специальной подготовки  
✅ **Интеграция с Authentik** — единая система управления пользователями  
✅ **Полнотекстовый поиск** — мгновенный поиск по всему контенту  
✅ **Аналитика** — отслеживание популярности и активности  
✅ **Docker Compose** — развёртывание в один клик

## 3. Интеллектуальный поиск

![Страница поиска](./skin_search_light.png)

### 3.1 Текстовый поиск

**Назначение:** Быстрый поиск по ключевым словам

**Как использовать:**
1. Откройте страницу поиска
2. Введите ключевые слова
3. Выберите режим "Текстовый поиск"
4. Система найдёт все релевантные страницы

**Примеры:**
- `API документация` — найдёт обе фразы
- `"REST API"` — точная фраза
- `документация -старая` — исключить слово

### 3.2 AI-поиск по смыслу

**Назначение:** Интеллектуальный поиск по содержанию

**Как использовать:**
1. Откройте страницу поиска
2. Введите вопрос или описание
3. Выберите режим "AI поиск"
4. Система найдёт релевантные страницы по смыслу

**Примеры:**
- `Как настроить аутентификацию?` → найдёт про login
- `Интеграция платёжных систем` → найдёт про платежи
- `Мониторинг и логирование` → найдёт про logs

### 3.3 Фильтры и автодополнение

**Доступные фильтры:**
- **Дата** — с какой по какую дату
- **Автор** — кто создал страницу
- **Статус** — черновик/опубликовано
- **Категория** — по тегам

**Автодополнение:**
- Система предлагает популярные запросы
- Подсказки на основе истории поиска
- Быстрый доступ к часто ищущимся страницам

### 3.4 Аналитика поисков

**Отслеживается:**
- Количество поисков
- Популярные запросы
- Время поиска
- Результаты поиска

**Использование:**
- Улучшение структуры контента
- Определение недостающей информации
- Оптимизация навигации

## 4. AI-помощник для написания

![AI-помощник](./skin_editor_light.png)

### 4.1 Генерация контента из плана

**Функция:** Создание полной статьи из краткого описания

**Использование:**
1. Откройте новую страницу
2. Нажмите "AI Assistant" → "Generate Content"
3. Введите тему или план
4. Система создаст полную статью

**Результат:**
- Введение
- Основные разделы
- Примеры и случаи использования
- Заключение

### 4.2 Расширение текста

**Функция:** Развёртывание кратких заметок в полноценные статьи

**Использование:**
1. Выделите текст
2. Нажмите "AI Assistant" → "Expand"
3. Система развернёт текст с деталями

**Результат:**
- Добавление контекста
- Развёртывание примеров
- Уточнение определений

### 4.3 Сокращение текста

**Функция:** Создание краткого резюме длинной статьи

**Использование:**
1. Выделите текст или нажмите на странице
2. Нажмите "AI Assistant" → "Condense"
3. Система создаст краткое резюме

**Результат:**
- Ключевые моменты
- Главные идеи
- Краткое описание

### 4.4 Изменение тона

**Доступные тоны:**
- **Формальный** — официальный стиль для документов
- **Неформальный** — дружелюбный, разговорный тон
- **Технический** — специализированный язык

**Использование:**
1. Выделите текст
2. Нажмите "AI Assistant" → "Change Tone"
3. Выберите нужный тон
4. Система переделает текст

### 4.5 Проверка грамматики

**Функция:** Автоматическое исправление ошибок

**Проверяется:**
- Орфография
- Пунктуация
- Грамматика
- Стиль

**Использование:**
1. Выделите текст или нажмите на странице
2. Нажмите "AI Assistant" → "Check Grammar"
3. Система найдёт и исправит ошибки

### 4.6 Исправление форматирования

**Функция:** Улучшение структуры и оформления

**Улучшает:**
- Структуру заголовков
- Списки и перечисления
- Отступы и выравнивание
- Цитаты и выделения

**Использование:**
1. Нажмите "AI Assistant" → "Fix Formatting"
2. Система улучшит оформление

### 4.7 Извлечение ключевых слов

**Функция:** Автоматическое определение главных терминов

**Использование:**
1. Нажмите "AI Assistant" → "Extract Keywords"
2. Система найдёт главные термины
3. Они будут добавлены как теги страницы

**Результат:**
- Автоматическая категоризация
- Улучшение поиска
- Связанные страницы

### 4.8 SEO-оптимизация

**Функция:** Рекомендации для лучшей видимости

**Предлагает:**
- Улучшение заголовков
- Добавление ключевых слов
- Оптимизацию описания
- Рекомендации по структуре

**Использование:**
1. Нажмите "AI Assistant" → "SEO Suggestions"
2. Система предложит улучшения
3. Примените рекомендации

### 4.9 Создание плана статьи

**Функция:** Генерация структуры статьи из содержания

**Использование:**
1. Нажмите "AI Assistant" → "Create Outline"
2. Система создаст план статьи
3. Используйте как основу для написания

**Результат:**
- Логичная структура
- Иерархия разделов
- Готовый план для развития

## 5. Управление доступом и группами

### 5.1 Роли пользователей

| Роль | Возможности |
|------|------------|
| **Администратор** | Полный доступ, управление системой |
| **Редактор** | Создание и редактирование страниц |
| **Читатель** | Только чтение назначенных страниц |
| **Гость** | Доступ только к публичным страницам |

### 5.2 Уровни доступа к страницам

- **Публичная** — видна всем пользователям
- **Для группы** — видна только членам группы
- **Приватная** — видна только владельцу

### 5.3 Управление группами

**Создание группы:**
1. Откройте админ-панель → Groups
2. Нажмите "Create Group"
3. Введите название и описание
4. Нажмите "Create"

**Добавление пользователей:**
1. Откройте группу
2. Нажмите "Add Member"
3. Выберите пользователя
4. Нажмите "Add"

**Установка прав:**
1. Откройте страницу
2. Нажмите "Share"
3. Выберите группу
4. Установите права (Read/Edit/Admin)
5. Нажмите "Save"

## 8. Развёртывание и инструкции

### 8.1 Требования

- Docker и Docker Compose (версия 1.29+)
- 8GB+ RAM (рекомендуется 16GB+)
- 20GB+ свободного места
- GPU (опционально)
- Authentik-сервер для OAuth2

### 8.2 Быстрое развёртывание

**Шаг 1: Клонирование**
```bash
git clone https://github.com/scoliologic/wiki.git
cd wiki
```

**Шаг 2: Конфигурация**
```bash
cp .env.example .env
nano .env
```

**Шаг 3: Запуск**
```bash
docker-compose up -d
```

**Шаг 4: Инициализация БД**
```bash
docker-compose exec app pnpm db:push
```

**Шаг 5: Загрузка AI-моделей**
```bash
docker-compose exec ollama ollama pull nomic-embed-text
docker-compose exec ollama ollama pull llama3.2
```

**Шаг 6: Доступ**
```
https://wiki.your-domain.com
```

### 8.3 Конфигурация окружения

```env
# Database
DATABASE_URL=postgresql://wiki:password@postgres:5432/wiki

# Authentik OAuth
AUTHENTIK_URL=https://authentik.your-domain.com
AUTHENTIK_TOKEN=your-api-token
AUTHENTIK_CLIENT_ID=your-client-id
AUTHENTIK_CLIENT_SECRET=your-client-secret

# Ollama AI
OLLAMA_URL=http://ollama:11434

# MinIO S3 Storage
MINIO_ROOT_USER=minioadmin
MINIO_ROOT_PASSWORD=minioadmin
MINIO_URL=http://minio:9000

# Application
JWT_SECRET=your-secret-key
VITE_APP_TITLE=Scoliologic Wiki
VITE_APP_LOGO=/logo.png
```

### 8.4 Обновление системы

```bash
git pull origin main
docker-compose down
docker-compose up -d
docker-compose exec app pnpm db:push
```
